# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in E-COS, please report it responsibly.

**Do not** open a public GitHub issue for security vulnerabilities.

Instead, email **ava.prime@outlook.com** with:

- Description of the vulnerability
- Steps to reproduce (PoC if possible)
- Potential impact
- Suggested fix (if any)

We will acknowledge receipt within 48 hours and aim to release a fix or mitigation plan within 7-14 days depending on severity.

## Scope

This policy covers the core E-COS Python package and its official documentation. It does **not** cover:
- Third-party dependencies (report to their maintainers)
- User-deployed instances or custom extensions
- Issues in downstream projects using E-COS

Thank you for helping keep E-COS and the broader Codessa ecosystem secure.
